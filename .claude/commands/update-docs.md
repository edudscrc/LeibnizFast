After implementing changes in the LeibnizFast codebase, update all relevant documentation files.

Steps:
1. Read the current `CLAUDE.md` to understand existing documentation structure.
2. Review the changes you just made (files modified, new modules, new APIs, changed signatures).
3. Update `CLAUDE.md`:
   - **File Structure** table: add/update entries for any new or renamed files.
   - **Key subsystems**: update descriptions if architecture changed.
   - **Key Patterns**: add new patterns introduced.
   - **Common Tasks**: add or update task guides if workflows changed.
   - **Build & Test**: update if build steps or test counts changed.
4. If new modules or significant features were added, ensure the **Architecture** section reflects them.
5. Do NOT add information that can be derived by reading the code — only document non-obvious architecture, patterns, and workflows.
6. Keep the documentation concise and scannable.
